Fantasy Baseball Draft Helper

View on GitHub

A local draft assistant with a custom-built recommendation engine that scores every available player in real time using a 10-factor composite model. Each factor is independently normalized to a common scale before weighting so no single component dominates the final score.

The core scoring pipeline starts with z-score valuation, calculating composite z-scores across all rotisserie categories (HR, OBP, R, RBI, SB for batters; ERA, K, SV, WHIP, WQS for pitchers) to measure how far each player deviates from the mean. Value Above Replacement (VAR) analysis then computes each player's surplus value relative to the best freely available player at their position. A Savant adjustment layer ingests Baseball Savant data to apply xStat-based corrections for players over- or under-performing their underlying metrics.

On top of the statistical core, the engine layers six additional factors: position scarcity (compares above-average remaining supply against league-wide demand with flex-position eligibility awareness), team needs (identifies unfilled roster slots and category gaps), relative advantage (uses full standings simulation to find categories where adding a player yields the most roto point movement), ADP value (tiered reach/steal penalties using logarithmic decay for picks far outside the relevance window), category balance with anti-punt protection (detects bottom-3 category rankings and applies escalating boosts to prevent double-punting), and opponent blocking (late-draft tiebreaker that flags players who would extend a rival's category lead).

The engine evaluates the top 150 players by ADP plus the top 20 available pitchers, filters for valid roster slots, and returns scored recommendations with full per-factor breakdowns. The Flask backend serves a real-time frontend with live draft tracking, search, filtering, and auto-saved draft state.

OpenClaw on Cloudflare

Deployed and configured OpenClaw, an autonomous personal AI agent, on Cloudflare's edge infrastructure. The agent runs inside a Cloudflare Sandbox container, an isolated ephemeral micro-VM that prevents RCE risk. Persistent memory and conversation history are stored in Cloudflare R2. The admin interface is locked behind Cloudflare Access with Zero Trust authentication (JWT validation, identity provider integration). Set up the Anthropic API key routing through Cloudflare AI Gateway for caching, analytics, and provider flexibility. Tuned the instance type and sleep timers for cost optimization. The entire setup runs on Cloudflare Workers for about $5/month with no VPS or local hardware required.

Fantasy Baseball Assistant

A full-stack in-season fantasy baseball intelligence platform built on top of a decade of historical data. The system ingests and indexes 10 years of standard stats, advanced metrics, multiple prediction models, player context (injuries, news, transactions), defensive ratings, park factors, pitcher-vs-hitter matchup splits, draft history, and league winner composition. All of this feeds a custom player/skill matchup engine that projects outcomes for every rostered and available player on a daily basis.

The matchup engine runs platoon analysis (L/R splits against opposing starters), closer projection modeling (role stability, leverage index, committee risk), and real-time context ingestion to track who is injured, slumping, or surging. It carries forward the same z-score, VAR, and category balance math from the Draft Helper, but layers on real machine learning via scikit-learn for in-season projections that adapt as the year progresses. The system surfaces actionable recommendations: who to pick up, who to drop, and potential trades that improve your category standings against the rest of the league.

The architecture is event-driven. A pub/sub model handles context and stat ingestion for in-season data as it arrives. API Gateway routes requests to Lambda functions that service the data layer. A graph database stores the relationship-heavy player, team, and matchup data. SQS queues handle async processing for batch projections and model retraining. The Go backend runs the core number crunching, projections, and scoring pipelines. OpenClaw runs on a Mac Mini as the orchestration and communication layer, delivering insights and recommendations directly to me. The Django frontend ties it all together with a Python/scikit-learn ML layer for model training, evaluation, and prediction serving.
